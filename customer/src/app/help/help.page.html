Operating systems

What is an Operating system:

An operating system is software that is used to control and manage the functionality of a computer. It functions as an interface for the computer’s hardware components together with some software for a user. This includes things like the CPU, RAM, and storage devices. The operating system ensures that each program gets access to the resources it needs to run, while also preventing any one program from hogging all the resources and causing the system to crash. Operating systems come in many different varieties, from Windows and macOS on desktop computers, to Android and iOS on mobile devices. Each operating system has its own unique set of features and capabilities, but they all share some basic functions.

How we set it up:

Once we have created and built our application system, we will make use of the most widely used computer operating systems, Microsoft Windows or iOS. This will allow almost anyone to make use of the system as our application will be compatible with most devices which mainly use either Microsoft Windows or iOS.


How it will work within the system:

Windows devices and Mac devices make use of these operating systems, therefore this will incorporate it with our system when it is used. It will support our web application. User interface: The user interface allows users to interact with the computer, either through a graphical user interface (GUI) or a command-line interface (CLI). The GUI provides a visual interface with icons, windows, and menus, while the CLI uses text-based commands to interact with the system Device drivers: Device drivers are software components that provide an interface between the operating system and hardware devices, such as printers, keyboards etc.. They allow the operating system to communicate with and control the hardware devices. File system: The file system organises and manages the computer's files and directories. It will allow users and our application to access and store data on the computer's storage devices, such as hard drives, solid-state drives, or network drives. Security features: Operating systems provide various security features to protect the computer and its data. These features include user accounts and permissions, firewalls, antivirus software, and encryption tools

Exporting from excel: SSIS

What is SSIS:
Exporting data from a system using SQL Server Integration Services (SSIS) is a powerful and flexible way to transfer data between different sources and destinations. SSIS is a data integration tool that is part of Microsoft's SQL Server suite of tools, and it allows you to extract, transform, and load (ETL) data from various sources into a destination system. How we set it up:
To export data from a source system using SSIS, we first need to create an SSIS package. This package typically contains one or more data flow tasks, which define the data flow from the source system to the destination system. Each data flow task contains one or more data flow components, such as sources, transformations, and destinations, that perform specific data transformation and manipulation operations. To create the SSIS package, we will use SQL Server Data Tools (SSDT), which is a free integrated development environment (IDE) for building SSIS packages. Once we have created the package in Visual Studio, we can configure the data flow tasks and components according to the system's specific requirements. How it will work within the system:
To export data from a SQL Server database, we will use the SQL Server Source component, which allows us to specify the SQL query that selects the data to be exported. We will then use various transformations, i.e., Data Conversion/Derived Column transformations, to transform and manipulate the data as needed. Finally, we will use the destination component (SQL Server) to write the data to the database. Once we have configured the SSIS package, we will deploy it to the SSIS catalog, which is a central repository for storing and executing SSIS packages. We can then execute the package manually or schedule it to run at specific times using SQL Server Agent, which is a built-in job scheduling and automation tool in SQL Server. In this system the client can export product details to an excel sheet. They can choose what tables and attributes from the database they would like to export and in what order the columns should be displayed. Why we think this is best:
Overall, exporting data from a system using SSIS is a powerful and flexible way to transfer data between different sources and destinations and it is already available in Visual Studio, which is one of the applications we will be using to create our system. With its rich set of data flow components and transformations, SSIS allows us to manipulate and transform data in many ways, and its scheduling and automation features make it easy to run the data export process on a regular basis.

Data layer:

What is a data layer:
In a system that uses API .NET Core and Angular workspace, a data layer plays a crucial role in managing the data flow between the client-side Angular application and the server-side .NET Core API. The data layer is a design pattern that separates the data access logic from the rest of the application, making it easier to manage and maintain data-related code. How we set it up:
To implement a data layer in a system that uses API .NET Core and Angular workspace, you can create a separate project for the data access logic. This project can include models for data entities, data access code using Entity Framework, caching code, and other related functionality. You can then use dependency injection to inject the data layer into your Angular service or .NET Core controller as needed. How it will work within the system: Abstraction of data access:
With a data layer, we can abstract the data access logic from the rest of the application. This means that the Angular application can interact with the data layer without worrying about the underlying data storage technology. The data layer can handle the specifics of interacting with the database, caching data, and other related tasks. Improved maintainability:
By separating data access logic into a separate layer, we can make the code easier to maintain and debug. Changes to the data access logic can be made in one place, without affecting the rest of the application. This makes it easier to test and update the application over time. Reduced duplication of code:
With a data layer, we can reuse data access code across multiple parts of the application. This can save development time and reduce the likelihood of errors in the code. The data layer can also help with code organisation, making it easier to find and reuse code. Improved performance:
By using a data layer, we can implement caching and other performance optimizations. The data layer can cache frequently accessed data, reducing the number of database queries needed to retrieve the data. Why we think this is best:
Overall, using a data layer in a system that uses API .NET Core and Angular workspace can improve maintainability, reduce duplication of code, and improve performance. By abstracting data access logic into a separate layer, you can make the application easier to manage and maintain over time
